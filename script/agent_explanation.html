🔬 试题优化 Agent 机制解析

一、什么是 Agent？

Agent（智能体） = 一个能感知环境、做出决策、执行动作的智能程序。

类比：就像一个智能助手，你告诉它你的目标，它会自动规划、执行、反馈。

在这个项目中：
用户 → 提交试题 → Agent → 评估 → 发现问题 → 优化 → 返回结果

四、工具系统（Tools）

Agent 内置了 5 个工具，就像一个工具箱：

show_status查看当前状态用户想看试题状态

怎么决定用哪个工具？
用户: "帮我优化这道题"
↓
Router 分析意图 → 这是在要求优化
↓
选择工具: optimize_question

五、评估器机制（3个指标）

每道试题会从 3 个维度 评估质量：

指标含义如何计算阈值
relevance（相关性）试题和知识上下文是否相关语义嵌入 + 余弦相似度>= 0.70
explainability（可解释性）答案是否唯一确定LLM 多次回答，看一致性>= 0.90
difficulty（难度）难度是否符合目标与题库同难度试题相似度>= 0.50
并行评估：3 个指标同时评估，不是一个一个来，速度更快！
示例：
一道题评估结果：
- relevance: 0.709 ✅
- explainability: 1.000 ✅
- difficulty: 0.500 ✅
平均分: 0.736 → 全部通过！

六、优化流程（迭代循环）

┌─────────────────────────────────────────┐
│           迭代优化循环                    │
│                                         │
│  Q0 (初始试题)                           │
│      ↓                                  │
│  ┌──────────────────────┐               │
│  │   并行评估 (3指标)    │               │
│  └──────────────────────┘               │
│      ↓                                  │
│  全部通过? ──是──→ 返回最佳试题           │
│      ↓                                  │
│   失败 N 个?                             │
│      ↓                                  │
│  N>=2 → 多目标优化                       │
│  N=1  → 单目标优化                       │
│      ↓                                  │
│  Q1 (优化后试题) ──→ 重新评估            │
│      ↓                                  │
│  最大 3 次迭代                           │
└─────────────────────────────────────────┘
关键点：
  • 最多迭代 3 次，防止无限循环
  • 记录每次迭代的最佳结果
  • 如果无法全部通过，返回历史最佳

七、意图识别（Router）原理

这是 Agent 的"大脑"，决定下一步做什么：

用户输入: "丰富一下这道题的描述"

↓
构建决策 Prompt:
  - 列出可用工具
  - 告诉 LLM 当前状态
  - 让 LLM 判断意图

↓
LLM 推理:
  "用户想要修改试题内容" → modify_question

↓
返回决策结果:
  {"tool": "modify_question", "args": {"instruction": "丰富描述"}}
本质：用 LLM 本身来做决策！让它自己决定该怎么做。

八、持续对话机制

不像简单的 Q&A，这个 Agent 可以记住上下文：

用户: 提交试题 A
Agent: 评估完成，3个指标全通过

用户: "把难度改成困难"
Agent: 调用 modify_question，调整难度

用户: "再优化一下"
Agent: 调用 optimize_question，重新评估

... 可以一直对话下去
关键设计：
  • current_question - 记住当前试题
  • conversation_history - 记住对话历史
  • 每次对话都知道上下文

十、总结：Agent 的三大能力

能力实现方式在这个项目中的体现
规划（Planning）LLM 推理决定下一步Router 分析意图选择工具
工具调用（Tool Use）调用外部函数执行动作评估、优化、修改工具
记忆（Memory）保存上下文状态current_question + history
🎉 学习要点：
Agent = LLM + Tools + Memory
通过prompt工程让 LLM 学会调用工具，
通过状态管理实现持续对话。
