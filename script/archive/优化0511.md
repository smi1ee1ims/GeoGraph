# 试题优化 Agent 开发会话总结

**日期**: 2026-05-11

---

## 需求概述

在已有的 graphRAG 试题生成工作流基础上，增加一个**试题优化 Agent**，能够：
1. 对生成的试题进行评估（3个指标）
2. 根据评估结果迭代优化试题
3. 支持持续对话，可响应用户追加消息
4. 智能路由，自动识别用户意图调用工具

---

## 完成的功能

### 1. 评估系统（3个指标）

| 指标 | 计算方法 | 阈值 |
|------|----------|------|
| relevance（相关性） | 语义嵌入 + 余弦相似度 | >= 0.70 |
| explainability（可解释性） | LLM 多次回答的一致性 | >= 0.90 |
| difficulty（难度准确性） | 与题库同难度试题的相似度 | >= 0.50 |

### 2. 评估函数集成

从 `eval.ipynb` 提取并集成了真实的评估函数：
- `compute_knowledge_relevance()` - 知识相关性
- `compute_explainability_single()` - 可解释性
- `eval_difficulty_accuracy_single()` - 难度准确性

### 3. 优化流程

```
Q0 → 并行评估(3指标) → 全部通过? → 返回
                          ↓ (否)
                    失败数 >= 2 → 多目标优化
                    失败数 = 1 → 单目标优化
                          ↓
                    Q{i+1} → 重新评估
                          ↓
                    最多3次迭代
```

### 4. 智能 Tool Router

用户输入自然语言，Agent 自动分析意图并调用对应工具：

| 工具 | 功能 | 触发场景 |
|------|------|----------|
| submit_question | 提交试题评估 | 用户提交试题 |
| optimize_question | 优化当前试题 | 用户要求优化 |
| modify_question | 按指令修改 | 用户说"改一下..." |
| show_status | 查看状态 | 用户想看状态 |
| chat | 聊天问答 | 其他对话 |

### 5. 持续对话机制

- 维护 `current_question` 状态
- 维护 `conversation_history` 对话历史
- 支持追加指令，如"丰富描述"、"改难度"等

### 6. 多 LLM 提供商支持

支持切换不同 API：
- deepseek（默认）
- anthropic
- minimax

## 使用方式

### 交互模式
```bash
python question_optimizer_agent.py --interactive
```

### 输入格式
```
题干;选项;答案;知识上下文;章节;难度;知识点
```

示例：
```
在我国西北干旱地区...被称为（    ）。;A. 风蚀柱 B. 风蚀蘑菇 C. 风蚀城堡 D. 风蚀洼地;B;风蚀地貌包含风蚀蘑菇。;风成地貌;简单;风蚀蘑菇
```

### 对话示例
```
用户: 帮我优化试题：<试题>
Agent: 评估 → 优化 → 返回完整试题

用户: 丰富一下这道题的描述
Agent: 调用 modify_question 修改

用户: 把难度改成困难
Agent: 调用 modify_question 调整难度
```

## 关键设计决策

1. **并行评估**：3个指标同时评估，提升效率
2. **迭代优化**：最多3次，追求全部通过指标
3. **返回最佳**：即使未全部通过，也返回历史最佳
4. **智能路由**：用 LLM 本身做意图识别决策
5. **知识点约束**：优化时强制要求围绕核心知识点
